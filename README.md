# cloud-grc-payload-workbench

24-week DevSecOps training workbook. No AI-written code.

## Structure

- `plan.md` — semester roadmap and weekly details
- `PARALLEL_TIMETABLE.md` — four-lane daily timetable for certs, tools, core skills, and consolidation
- `study-cockpit.html` — standalone interactive cockpit with local progress storage
- `weeks/` — weekly workbooks (theory + tasks + review)
- `drills/` — hands-on practice by topic
- `trackers/` — cert sprint, control mapping, monthly artifacts, and nightly logs
- `AI_BOUNDARY.md` — what AI can and cannot do here
- `WEEKLY_RUBRIC.md` — end-of-week grading criteria

## Week 1

**Topic:** Bash fundamentals and CI script safety
**Folder:** `weeks/week-01-bash-ci-safety/`
**Artifact:** one safe Bash script with clear failure behavior

## Rules

- AI explains concepts and grades rubrics only
- AI does not write, patch, or debug your code
- Commit often with messages that cite the risk, control, or gap

## Current Operating System

Use `PARALLEL_TIMETABLE.md` as the active study rhythm:

- Morning: certification lane
- Afternoon: variable tooling/security lab
- Evening: core skills grind
- Night: consolidation and evidence mapping
