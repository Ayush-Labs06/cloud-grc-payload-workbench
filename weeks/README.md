# Weekly Workbook

Each week is a self-contained training unit. Use the folder for that week as the working record of what you planned, built, broke, understood, and reviewed.

## Required Files Per Week

- `README.md` - week goal, artifact, and done condition.
- `theory.md` - concepts to understand before coding.
- `tasks.md` - manual checklist derived from the roadmap.
- `debuglog.md` - real errors and reasoning trail.
- `evidence.md` - proof of work and artifact references.
- `review.md` - end-week reflection and rubric result.

Use `_template/` when starting a new week.
